# Design Uber Ride Matching

## Problem Statement

Design a ride-sharing service like Uber. Riders request rides, drivers accept them, and the system matches them efficiently based on proximity and other factors.

---

## 1. Requirements

### Functional Requirements

| Requirement | Description |
|-------------|-------------|
| **Request ride** | Rider requests ride with pickup/dropoff |
| **Match driver** | Find and match nearby available driver |
| **Track ride** | Real-time location updates during ride |
| **Fare calculation** | Calculate and charge fare |
| **ETA** | Show estimated time of arrival |

### Non-Functional Requirements

| Requirement | Target |
|-------------|--------|
| **Latency** | Match driver in < 30 seconds |
| **Availability** | 99.99% uptime |
| **Scale** | 10M rides/day globally |
| **Location accuracy** | Update every 4 seconds |

---

## 2. Capacity Estimation

### Traffic

```
RIDES:
- 10M rides per day
- 10M / 86400 вүҲ 115 rides/second average
- Peak (rush hour): 500 rides/second

LOCATION UPDATES:
- 1M active drivers
- Update every 4 seconds
- 1M / 4 = 250,000 location updates/second
```

### Storage

```
RIDES:
- 10M rides/day Г— 1 KB per ride = 10 GB/day
- 1 year: ~3.6 TB

LOCATION HISTORY:
- 250K updates/sec Г— 100 bytes = 25 MB/second
- Keep 30 days: ~65 TB
```
