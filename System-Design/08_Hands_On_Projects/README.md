# Hands-On Projects

> "Theory without practice is useless. Practice without theory is blind."

Build these mini-projects to solidify your understanding of core system design concepts.

## Why Build Projects?

```
┌────────────────────────────────────────────────────────────────┐
│                  LEARNING BY BUILDING                           │
├────────────────────────────────────────────────────────────────┤
│                                                                 │
│  READING ABOUT CACHING:                                        │
│  "LRU evicts the least recently used items..."                 │
│  Understanding: 60%                                            │
│                                                                 │
│  IMPLEMENTING LRU CACHE:                                       │
│  "I need a doubly linked list for O(1) removal,               │
│   and a hashmap for O(1) lookup..."                           │
│  Understanding: 95%                                            │
│                                                                 │
│  Building forces you to understand the details!                │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

## How to Use These Projects

```
FOR EACH PROJECT:

1. READ the problem description
2. TRY implementing yourself first (30-60 min)
3. COMPARE with provided solution
4. RUN and test the code
5. TRY the extensions

Don't just copy-paste! The learning is in the struggle.
```
