## The Interview Reality

```
┌─────────────────────────────────────────────────────────────────────┐
│                WHAT ACTUALLY HAPPENS                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Before Interview:                                                 │
│  - You get a vague problem: "Design Twitter"                       │
│  - No hints about what aspects to focus on                         │
│  - Interviewer watches how you handle ambiguity                    │
│                                                                     │
│  During Interview:                                                 │
│  - You drive the conversation (interviewer mostly listens)         │
│  - You must ask clarifying questions                               │
│  - You draw diagrams while explaining                              │
│  - Interviewer may redirect or ask follow-ups                      │
│                                                                     │
│  What's Evaluated:                                                 │
│  - Problem-solving approach (not just the solution)                │
│  - Communication clarity                                           │
│  - Technical depth                                                 │
│  - Handling trade-offs                                             │
│  - Response to feedback                                            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Quick Tips

### Do's
- ✅ Ask clarifying questions before designing
- ✅ State your assumptions explicitly
- ✅ Start with high-level design, then dive deep
- ✅ Discuss trade-offs for every major decision
- ✅ Use concrete numbers (capacity estimation)
- ✅ Acknowledge what you don't know

### Don'ts
- ❌ Jump straight into solution
- ❌ Stay silent while thinking
- ❌ Ignore interviewer's hints
- ❌ Over-engineer simple problems
- ❌ Skip capacity estimation
- ❌ Forget about failure scenarios
