## 🎯 Problem-Solving Template (For Each Problem)

### Step 1: Understand (5 mins)
- [ ] Read problem 2-3 times
- [ ] Understand constraints
- [ ] Identify input/output format
- [ ] Note examples

### Step 2: Think (15-20 mins)
- [ ] Draw diagrams/examples
- [ ] Think of 2-3 approaches
- [ ] Identify better approach
- [ ] Estimate time/space complexity

### Step 3: Code (20-25 mins)
- [ ] Write pseudocode
- [ ] Implement solution
- [ ] Add comments
- [ ] Clean up code

### Step 4: Test (5-10 mins)
- [ ] Test on provided examples
- [ ] Test edge cases
- [ ] Verify time/space complexity

### Step 5: Optimize (5-10 mins)
- [ ] Can it be faster?
- [ ] Can it use less memory?
- [ ] Better approach?

### Step 6: Learn (5 mins)
- [ ] Look at other solutions
- [ ] Learn different approaches
- [ ] Note patterns

## 🆘 If You Get Stuck

### Stuck for 15 minutes?
1. Reread the problem
2. Draw a diagram
3. Think of edge cases

### Stuck for 25 minutes?
1. Look at the hint
2. Read similar problem from earlier
3. Look at approach (not code)

### Stuck for 40 minutes?
1. Look at the code solution
2. Understand each line
3. Rewrite it yourself

### Stuck for 60 minutes?
1. Mark as "need review"
2. Move to next problem
3. Return after 2-3 days

## 💪 Mental Health Tips

**When you feel stuck:**
- "This is normal, everyone struggles"
- "I'm learning, not failing"
- "Break it into smaller parts"

**When you're going fast:**
- "Don't skip the understanding phase"
- "Quality over speed"
- "Can you explain it to someone?"

**When you're tired:**
- Take a break (seriously!)
- Sleep > grinding when tired
- Come back with fresh eyes
