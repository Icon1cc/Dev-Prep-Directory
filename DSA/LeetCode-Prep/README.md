## 🎯 How to Use This Program

### Step 1: Set Up Your Environment

1. Fork or download this repository
2. Open `progress_tracker.xlsx` to track your daily completion
3. Bookmark: https://leetcode.com/

### Step 2: Daily Workflow

**Each day, follow this pattern:**

1. **Read the Day's Overview** (`day_XX.md`)
   - Review topics for the day
   - Understand learning objectives

2. **Solve 6-8 Problems**
   - Look at the daily file for problems assigned to that day
   - Or reference `leetcode_master_sheet.csv` for Day column
   - Start with Easy problems, progress to Medium/Hard

3. **For Each Problem:**
   - Read the problem statement carefully
   - Think about the approach for 15-20 minutes
   - Write code (try to solve without looking at solutions first)
   - Test edge cases
   - Check your solution against examples
   - Read other solutions if stuck

4. **Track Progress**
   - Mark problem as "Completed" in `progress_tracker.xlsx`
   - Add notes about your approach
   - Time how long you spent

5. **Daily Reflection**
   - Review key concepts at end of day
   - Note patterns you learned
   - Identify weak areas for review

### Step 3: Weekly Review

Every 7 days:
- Review problems from that week
- Re-solve problems you found difficult
- Make sure you understand the intuition, not just the solution

### Step 4: Monthly Deep Dive

At day 15, 30, and 45:
- Re-solve 5-10 hardest problems from the period
- Practice explaining your solution out loud
- Time yourself (simulate interview conditions)

## 💾 Using the Master Data Files

### CSV Format (`leetcode_master_sheet.csv`)

Columns:
- **Day**: Day number (1-45)
- **Problem Number**: LeetCode ID
- **Problem Name**: Full problem title
- **LeetCode Link**: Direct link to problem
- **Difficulty**: Easy / Medium / Hard
- **Topic**: Main category (Arrays, Trees, DP, etc.)
- **Subtopic**: Specific pattern
- **Pattern**: Algorithm pattern used
- **Companies**: Companies that ask this
- **Hint**: One-line hint for solving
- **Completed**: Mark TRUE when solved
- **Notes**: Your personal notes

### Excel Format (`leetcode_master_sheet.xlsx`)

**Features:**
- Filter by Day, Topic, Difficulty, Pattern, or Company
- Color-coded difficulty levels
- Track completion status
- Add personal notes

**How to use:**
1. Open in Excel/Google Sheets
2. Use Data > AutoFilter
3. Filter by your preferred columns
4. Sort by company if preparing for specific interviews

## 🚀 Best Practices

### DO ✅

- ✅ **Solve without solutions first** - Try for 20-30 mins before checking
- ✅ **Understand the intuition** - Not just memorizing code
- ✅ **Optimize iteratively** - Start with brute force, then optimize
- ✅ **Explain out loud** - Practice articulating your approach
- ✅ **Test edge cases** - Empty arrays, single elements, large inputs
- ✅ **Time yourself** - Simulate interview pressure
- ✅ **Review solutions** - Learn from optimal approaches
- ✅ **Note patterns** - Recognize similar problems
- ✅ **Practice consistently** - Daily habit > sporadic marathon sessions

### DON'T ❌

- ❌ **Don't just read solutions** - You won't retain knowledge
- ❌ **Don't skip easy problems** - Foundation is crucial
- ❌ **Don't solve in isolation** - Explain your approach to others
- ❌ **Don't memorize code** - Understand the algorithm
- ❌ **Don't rush through** - Quality over quantity
- ❌ **Don't ignore weak areas** - Address gaps immediately
- ❌ **Don't stop at one solution** - Explore multiple approaches
- ❌ **Don't skip hard problems** - Push your boundaries

## 🔄 When You Get Stuck

### Step 1: Think (15-20 mins)
- Break problem into smaller parts
- Draw diagrams
- Consider multiple approaches
- Think aloud

### Step 2: Hint
- Check the "Hint" column in master sheet
- Review similar problems solved earlier
- Look at related topic discussion

### Step 3: Partial Solution
- Look at approach, not code
- Read problem discussion on LeetCode
- Find similar problem discussion

### Step 4: Code Review
- Check solution code
- Understand each line
- Rewrite without looking

### Step 5: Practice Similar
- Solve 2-3 similar problems
- Practice the pattern
- Build confidence

## 🎤 Interview Preparation Tips

### 1. Behavioral + Technical
- 50% technical skills from this program
- 50% behavioral (teamwork, leadership, conflict resolution)
- Practice behavioral questions separately

### 2. Mock Interviews
- Week 6: Start mock interviews
- Week 8-9: Do 1-2 mocks per week
- Use Pramp, Interviewing.io, or peer practice

### 3. Communication
- Explain your approach before coding
- Ask clarifying questions
- Discuss tradeoffs
- Think out loud

### 4. Practice Explanations
- Can you explain a problem in 2 minutes?
- Can you explain your approach before coding?
- Can you walk through an example?

### 5. System Design (Weeks 8-9)
- This program focuses on algorithms
- After this, learn system design separately
- Start with basics (Grokking System Design)
