## Daily Routine (built for pattern retention)

1. **Pattern recall (10 min):** Before opening any new problem, write down — from memory — the patterns from yesterday's topic and one example each. If you can't, redo yesterday's hardest problem first.
2. **New problems (90–120 min):** ⭐ first, then by difficulty.
3. **Same-day re-solve:** Every problem you used a hint on, solve again from scratch the same evening. No exceptions.
4. **Spaced review (20 min):** Re-do 1–2 problems from 3 days ago.
5. **End-of-day note:** One line per problem in `LeetCode-Prep/progress_tracker.csv` — Solved / Hint / Re-do — and the **pattern name** (e.g. "two-pointer in-place", "prefix sum + hash", "Floyd's cycle").

## Rules

- Time-box: 30 min Easy, 45 min Medium. After that, read the editorial — then rewrite from scratch the next day.
- A problem isn't "done" until you've solved it cleanly without any reference **and** can name the pattern.
- After each topic, write a 1-page cheat sheet: when-to-use template, edge cases, the trickiest problem from the section. This is the artifact that fixes forgetting — refer to it before each new topic.
